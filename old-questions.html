Multiprogramming Spooling in operating system

  • In some devices where the peripheral are slower Input devices gives the process to the CPU during the time of transfer from input devices to CPU, the CPU is in idle state.
  • Also after the execution of the process , the sending of output data from CPU to output devices also take time during which the CPU is again idle.
  • In that way the CPU is in idle state most of the time, which is bad for any operating system design.
  • Hence the concept of spooling was introduced in which we store the data first in a memory (such as buffer, I/O device-specific interrupts, etc) so when the CPU requests for the process, it can transfer faster meaning CPU idle time is lesser.
  • Spooling is a specialized form of multiprogramming for the purpose of copying data between different devices.
  • It is usually used as a mediater between two devices with different speed flow, for example: printers, While the computer produces the files to be printed fast the printer is slower than any CPU hence a SPOOL is used to store the files which will be later used by the printer to print.

Critical section and minimum requirements for a solution of Critical section problem

  • The critical section is a code segment where the shared variables can be accessed. That resource may be any resource in a computer like a memory location, Data structure, CPU or any IO device.
  • Minimum requirements are:-
    1. Mutual Exclusion When one process is executing in its critical section, no other process is allowed to execute in its critical section.
    2. Progress When no process is executing in its critical section, and there exists a process that wishes to enter its critical section, it should not have to wait indefinitely to enter it.
    3. Bounded waiting There must be a limited waiting time for each process.

Semaphores

  • Semaphores are the interger variable used as a synchronizing tool for managing concurrent processes.
  • Properties of semaphores are:-
    • It has a non-negative value
    • They are machine independent
    • Each critical section has unique semaphores
  • Used for solving synchronization problem
  • Used for achieving mutual exclusion
  • Limitations are:-
    • They are global variables
    • No linguistic connection between semaphore and data the process holds.
