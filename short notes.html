Feedback system

  • Feedback system is those system which has input coupled with output, i.e the input takes the values from output as a feedback. The output of the system is again connected back to the input of the system for supplying the next set of inputs.
  • This kind of system is used in many scenarios like: aircraft system, error correction mechanism, etc where the output or result of the system is coupled back to the input and sent to the system again.
  • A very simple example of a feedback system is a heating system controlled by thermostat.

  3. It allows us to control time hence we can operate the system against several months or years of experience in a matter of seconds.
  4. It allows us to gain insights into how a model the system actually works and understanding of which variables are most important up to a performance.

Distributed Lag Model

  • Those models which base the current values of the system variables with other current values or the previous values of the system are called as distributed lag model.
  • For example:
    • If we have say 4 varaiables c, y, t, i
    • and the relation is as follows
      • c = 20.0 + 6.6*y
      • y = 12.0 - t
      • t = 8.0*i
      • i = c + y - t
    • The variables each are interconnected with other variables which means for a example the variable c must have first the variable y as well as y must have t hence the values of the variables are based upon the values of other variables.
    • We can also make this static model into dynmaic by lagging each variables to the previous values.
    • Say
      • c = 20.0 + 6.6*y-1
      • y = 12.0 - t-1
      • t = 8.0*i-1
      • i = c-1 + y-1 - t-1
    • With initial value given we can find the future values of each variables.
  • It is extensively used in econometric studies.

Cobweb model

  • Cobweb model is a distributed lag model used in ecomonic studies of price changing beign subject to the periodic fluctuations in certain market.
  • It is called a cobweb model as the graph for supply and demand usually forms a cobweb (spider web) like structure.
  • There are 2 most basic outcome of a cobweb model
    1. Convergent case : If the supply curve is steeper than demand curve, then the fluctuations decrease in magnitude with each cycle, creating a inward spiral for price and quantities.
    2. Divergent case : If the demand curve is steeper than supply curve, then the fluctualtions increase in magnitude with each cycle, creating a outward spiral for price and quantities.
  • This model is specially effective in businesses where the time between demand and supply is lagging which means the supply will be done after a lag or delay from the initial demand.
  • One such case is agricultural sector where planting and harvesting is done after some lagging or delay. With each year farmer planting the crop as much as required based in the previous year.

Measuring queuing system

  • Queuing system is a simulation of a queue in any system. It is common for many systems to have a queue model so that customer can wait in the queue before their time to get served by the system.
  • Queuing system is a must have system in many service based systems like supermarket, barbershop, online form, etc but we have to measure the queuing system and optimize it if not then the following problem might occur:-
    1. Customer waiting in queue for too long
    2. Server taking too much time to serve a single customer
  • Hence we measure the queuing system by following factors:-
    1. Average number of customer in queue in the system: By calculating/measuring the average number of customer in the queue, we can calculate the queue length and figure out the limit for queue lengths.
    2. Average waiting time in queue : Customers doesnot want to wait for too long hence measuring average waiting time can give an insights of how much time a customer is waiting and what kind of queuing system should be implemented to the system.
    3. System utilization : Having the server idle is not too good either for most cases and hence system utilization is measured to see how much the service or server is being used in the system. It is calculated by ratio of average arrival rate (ƛ) to average service rate (μ)

Components of queuing system

  • Queuing system is a simulation of a queue in any system. It is common for many systems to have a queue model so that customer can wait in the queue before their time to get served by the system.
  • There are many types of queue system that servers the customer in different queuing disciplines:-
    • First In First Out
    • Last In First Out
    • Priority based, and many more
  • But for a queuing system to be complete in any format it must have the following components:-
    1. Calling Source : The source from which the customers are coming from or the customers are generated from is called calling source. It can be infinite like patient arriving at hospital, or finite like a repair crew with only 5 machines to look after.
    2. Queue : Waiting line or queue is the actual space where the customer wait before their service time is arrived or their turns arrives.
    3. Server : Server or service facility is the point at which the service is provided to the customers after the queue. Depending upon the system it can be a single server or multiple server serving the customers.

Properties of Random numbers

  • Random numbers are the numbers drawn from a uniformly distributed random variables between specified intervals and each having equal probability of occurance.
  • It is useful in many simulation models like Monte Carlo, to make the simulation either as free of bias as possible or introducing some random activities in the simulation.
  • For a number to be random number it must be from a distribution of random variables which all has the following properties:-
    1. Uniformity
      • For the random variables to be randomly drawn from the distributed set it must be uniform, i,e the difference between each random variables must be uniform in order to have a equal probability of drawing each random varaible from the uniformly distributed set of random varaibles*
    2. Independence
      • Independence of a random variable defines that a random varaible must not affect the outcome of another random variable, i.e if a random number day 0.22 is drawn in between 0 to 1 then in the second draw it shouldnot affect the probability of drawing other numbers or itself

Stochastic activities

  • Stochastic activities are those activites that have a variable outcome for the given input set. i.e unlike a deterministic activity where the input determines the outcome, here the input doesnot determine the outcome but is random
  • For example in a repair shop the number or machines repaired from hour to hour basis is in a random fashion. Some hour it may be 2, some hour may br 5, and so on with out any deterministic factor.

Uniformity test

  • Uniformity test is the process of validating a random number generater if it uses a uniformly distrubuted random variables or not.
  • Randon numbers must have the property of uniformity satisfied, i.e the probability of drawing each random varaible must be uniform or equal
  • There are two methods available for the uniformity test KS test and Chi-square Test both of which compares the generated random numbers with the theoritical uniform distribution.
  • It is a must do validation test for any random number generator as if we use a random number generator that hasn’t passed the uniformity test then we might get the same number more than other random numbers.

SSQM

  • SSQM (Single server queuing model) is a type of queuing model which has only single server to serve the queue in the system.
  • In Kendall Notation it can be represented as M/M/1 or M/M/1/∞\infin∞/FIFO
  • which means the arrival time and service time is exponention and only 1 server with ∞\infin∞ capacity and a FIFO discipline
  • Some examples of SSQM are: Drive through ordering system in resturants, patients going to visit a specific doctor, etc all of this system has only one server, in case of hospital only one doctor and in case of resturants only one window to serve the customers and patients.
  • Advantage of using SSQM is, it is simple to implement and understand
  • Disadvantage of using SSQM is, for larger number or customers and higher arrival rate than service rate the queue can be lengthy and average waiting time is longer.

Arrival Patterns in discrete simulation

  • The generation of exogenous arrival is an important aspect of discrete system simulation. An exact sequence of arrivals may have been specified for the simulation. For example, in the simulation of the electronic circuit, a particular sequence of signal might be used as the simulation input to verify the circuit.
  • The patterns are:-
    1. Trace Driven simulation:
      • It is the process of gathering a sequence of inputs based on the observation of a system. When there is no interaction between exogeneous arrival and the endogeneous events of a system, it is premissible to create a sequence of arrival in preparation for the simulation. Usually however the simulation proceeds by creating new arrivals as they are needed.
    2. Bootstrapping
      • It is the process of making one entity creates its successor. Those methods require keeping only the arrival time of the next entity, it is therefore the preferred method of generating arrivals for computer simulation programs.
