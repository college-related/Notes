  • Java is a platform independent language/ machine independent/ architectural neutral.

    • Java is a platform independent langaugae or architectural neutral langugae because the class file which is generated by JRE can be used to run the application in any OS. (i.e if the class file was generated by JDK from a windows system then it can be used to run the application in apple’s system or linux)
    • Unlike other programming langauge like C, C++, etc whose .exe file generated from one OS cannot be used in another OS, the .class file of Java can be used to run in every OS.
    • This is possible by the JVM(Java Virtual Machine) that is installed with the JDK which converts the .class file into the corresponding machine code to run.
    • Hence Java is a platform independent langauge or machine independent or architetural neutral.

  • How JVM makes Java platform independent language.

    • JVM(Java Virtual Machine) is one of the technology of Java that is installed along with other technologies like JRE(Java Runtime Environment), JIT(Just In Time) Compliation, etc when the JDK(Java Development Kit) is installed.
    • The JVM itself is a machine dependent technology, which means the corresponding JVM is installed to the corresponding OS. By doing so the JVM can take the .class file generated from any machine, any OS and convert it into the corresponding machine code.
    • Hence making the Java langauge a platform independent language.

Constructor

  • Definition, Usage and Type

    • Constructor is a special type of method function of a class that doesn’t have any return value and have same name as the class name. It doesn’t need any explicit invoking like any other methods, as it runs when a new instance of that class is created.
    • It is mostly used to initialize the data members/instance variables with initial values.
    • There are 3 types of constructor in java:-
      1. Default Constructor
        • It is the constructor which doesn’t have any parameters.
      2. Parameterized Constructor
        • It is the constructor which have parameters.
      3. Object Constructor
        • It is the constructor which takes the object of the same class as the parameter.

  • Example Code


    class ConstructorDemo {

        int num;

        ConstructorDemo(){
            num = 0;
            System.out.println("Default Constructor");
        }

        ConstructorDemo(int n){
            num = n;
            System.out.println("Parameterized Constructor");
        }

        ConstructorDemo(ConstructorDemo ob){
            this.num = ob.num;
            System.out.println("Object Constructor");
        }

        public static void main(String [] args){
            new ConstructorDemo();
            ConstructorDemo ob1 = new ConstructorDemo(1);
            ConstructorDemo ob2 = new ConstructorDemo(ob1);
        }
    }

    /*
    Output -
        Default Constructor
        Parameterized Constructor
        Object Constructor
    */

Inheritance

  • Definition

    • Inheritance is a feature of an OOP language that allows us to inherit data members as well as methods from one class to another same as childrens inheriting from thier parents.

  • Types

    • In java we have 3 types of inheritance. They are:-
      1. Single Inheritance
        • It is the simplest type of inheritance where one child class extends from one base class.
      2. Multilevel Inheritance
        • It is the type of inheritance where child class of one base class is the base class of another class and so on.
      3. MultiPath Inheritance
        • It the type of inheritance where one base class is inherited by two or more child classes.

  • Usage

    • Inheritance is used in java for:-
      • Code reusability, the same class can be extended to many different classes which makes the code reusable.
      • DRY code, by using inheritance we can gain the DRY(Do not Repeat Yourslef) codes, as we don’t have to repeadtly rewrite the instance variables and methods.

  • Multiple inheritance is not supported in Java?

    • Multiple inheritance is the type of inheritance where more than one base class is extended by the same single child class.
    • When inheriting from a base class all the methods and instance variables of that class is inherited to the child class, So if two base classes have methods with same definition than when trying to access that method through the child class there may occur ambiguity.
    • In C++ there are methods like virtual function or virtual base class or scope resolution to solve that ambiguity but using them we might loose some security points or loose the originality of the base class.
    • Hence in java the support for multiple inheritance is not present.
        class ChildClass extends BaseClassOne, BaseClassTwo {
        //...
    } 
    // illegal

  • Example


    class BaseClass {
        public void show(){
            //...
        }
    }

    //Single inheritance
    class ChildClass extends BaseClass {
        public void dis(){
            //...
        }
    }

    //MultiPath inheritance
    class ChildClassTwo extends BaseClass {
        //...
    }

    //Multilevel inheritance
    class ChildClassLevelTwo extends ChildClass {
        public static void main(String [] args){
            //...
        }
    }

Polymorphism

  • Definition

    • Polymorphism is a feature of an OOP langauge that allows us to make methods or functions that have same definition but different implementation. (i.e one function can have different functionality)

  • Types

    • There are 2 types of polymorphism in java:-
      1. Compile-Time Polymorphism
        • It is the polymorphism gained during the compilation time.(i.e the functions will be pre determined in which case which function to run during compilation)
        • Method overloading is a example of Compile-Time Polymorphism.
      2. Run-Time Polymorphism
        • It is the polymorphism gained during the runtime.(i.e the functions are determined in which case to use which function during the runtime)
        • Method overriding is a example of Run-Time Polymorphism

  • Usage

    • Polymorphism is used in java for:-
      • Generic Program, by using polymorphism we can create methods that works with more than one data type making the program generic.
      • We can create Generic api’s that can handle different data types.

  • Example


    class BaseClass {
        public void show(){
            System.out.println("This is in base class");
        }
    }

    class PolymorphismDemo extends BaseClass {
        
        //method overriding
        @Override
        public void show(){
            System.out.println("This is overrided");
        }

        //method overloading
        public void add(int n1, int n2){
            System.out.println("Sum is "+(n1+n2));
        }

        public void add(double n1, double n2){
            System.out.println("Sum is "+(n1+n2));
        }

        public static void main(String [] args){
            PolymorphismDemo obj = new PolymorphismDemo();

            obj.show(); // This is overrided
            obj.add(1, 4); // Sum is 5
            obj.add(3.5, 6.5); // Sum is 10.0
        }
    }

  • Can Static and Private methods overriden?

    • No, we cannot override a private or static method.
    • This is because;-
      • Static method are bonded during compile time and the reference to that method is only one made by the compiler, hence overridding only hides the new static method and calls the compiled one.
      • Private method cannot be inherited, hence trying to override a private method throws an compile time exception.
    • Though static and private methods can be overloaded
    class One {
        private void mssg(){
            System.out.println("This is a private method.");
        }

        public static void show(){
            System.out.println("This is a static method.");
        }
    }

    class PrivateStatic extends One {

        private void mssg(){
            System.out.println("Trying to override private method.");
        }

        public static void show(){
            System.out.println("Trying to override static method.");
        }

        public static void main(String[] args) {
            One obj = new PrivateStatic();

                obj.mssg();//throws an error as the method is private and cannot be accessed from derived class.

    }

Abstract

  • Definition and Usage

    • Abstract is a keyword used in Java to make a class or method abstract.
    • A abstract method is that method which have only method definition and doesn’t have any implementation(body part).
    • A abstract class is that class which consists of at least one abstarct method in it. It can have normal methods and instance variable as well. Though a object of an abstract class is not possible.
    • It is used for:-
      1. Gaining Abstraction
      2. Making a class or method abstract
      3. Making a blueprint base class for other classes

  • Example


    //Abstract class
    abstract class AbsDemo {
        public int num = 5;

        //Abstract method
        abstract public void show();

        public void normal(){
            System.out.println("It can have normal methods too");
        }
    }

    class Demo extends AbsDemo {

        @Override
        public void show(){
            System.out.println("We should override the abstract methods");
        }

        public static void main(String [] args){
            Demo d = new Demo();

            d.show();
            d.normal();
            d.num = 3; 
        }
    }

Package

  • Definition and Usage

    • Package is a feature of Java language that allows us to create folders that can act as packages to hold classes.
    • There are Built-in and User-defined packages in java.
      • The Built-in package are those packages which are pre written classes that are free to use, included in the Java Development Environment.
      • User-Defined packgae are those packages which are written and defined as package by User/Programmer ourselves.
    • Package is used in java for:-
      1. To make packages that holds classes with similar features, hence making the project folder more maintainable
      2. Used to make classes with same name in different packages in order to avoid name conflicts.
      3. By making a class in a package we can import it in any other classes to reuse them.

  • Example


    import java.awt.*; // A built-in java package

    class Demo extends Frame {
        Demo(){
            setSize(200, 200);
            setVisible(true);
        }

        public static void main(String [] args){
            new Demo();
        }
    }
  • To make your own user-defined package

    //Inside a folder say named myPack
    //Create a file named CustomPackage
    //at the top define the class as package

    /*----CustomPackage.java----*/
    package myPack; // the folder name

    public class CustomPackage {
        public void someFunction(){
            //...
        }
    }

    //Now outside of the folder myPack
    //Inside any other file import the CustomPackage

    import myPack.CustomPackage;

    public class Demo extends CustomPackage {
        public static void main(String [] args){
            Demo d = new Demo();

            d.someFunction();
        }
    }

Reflection

  • Definition

    • Java Reflection is a process of examining or modifying the run time behavior of a class at run time.
    • The java.lang.Class class provides many methods that can be used to get metadata, examine and change the run time behavior of a class.
    • The java.lang and java.lang.reflect packages provide classes for java reflection.
    • The Reflection API is mainly used in:
      1. IDE (Integrated Development Environment) e.g. Eclipse, MyEclipse, NetBeans etc.
      2. Debugger
      3. Test Tools etc.
    • Some of the methods are:-
      1. forName()
      2. getName()
      3. isInterface()
      4. isPrimitive()

  • Example


    class Test {}
    interface TestTwo {}

    class Demo {
        public static void main(String [] args){
            Class c1 = Class.forName("Test");
            Class c2 = Class.forName("TestTwo");

            System.out.println("Class name of c1 is "+c1.getName());
            System.out.println("Class name of c2 is "+c2.getName());
            System.out.println("Is c2 a interface =  "+c2.isInterface());
        }
    }

Access modifiers

  • Types and Usage

    • In Java the classes have two access modifiers:-
      1. Public
        • Classes with public access can be used everywhere as long as imported.
      2. Default
        • Classes with Default access can be used only inside the same package as the class defined.
    • For methods and Instance variables there are four access modifiers:-
      1. Public
        • Public methods and instance variables can be used everywhere with the object of that class.
      2. Protected
        • Protected methods and instance variables can be used everywhere with the object of that class only if that class is inherited.
      3. Private
        • Private methods and instance variables can be used only with in the class.
      4. Default
        • Default methods and instance variables can be used everywhere inside the same package as the class with the object of that class.
    • Access Modifiers are used in java for:-
      1. Modifiying the access level of classes, methods and instance variables.
      2. For gaining encapsulation
      3. By making methods and instance variables private we can make more secure programs.

  • Example


    //It can be used everywhere if imported
    public class PublicClass {
        //This function can be used everywhere if object of this class is made
        public void publicFunction(){}
        
        //This function can be used everywhere if this class is inherited or only in this class
        protected void protectedFunction(){}
        
        //This function can be used only inside this class
        private void privateFunction(){}

        //This function can be used everywhere in current package with the object of this class but not outside the package
        void defaultFunction(){}
    }

    //It cannot be used outside this package
    class DefaultClass extends PublicClass {
        public static void main(String [] args){
            DefaultClass ds = new DefaultClass();

            ds.defaultFunction();
            ds.publicFunction();
            ds.protectedFunction();
        }
    }

    class AnotherClass {
        public static void main(String [] args){
            PublicClass obj = new PublicClass();

            obj.publicFunction();
            obj.defaultFunction();
           
           // obj.protectedFunction(); error
        }
    }

Instance Varibale hiding

  • In Java we can create variables with same name as long as they have different scope levels.
  • We can create a instance variable named n and a local variable inside a method named n.
  • And when trying to access the n variable inside the method then the instance variable n will be hidden by the local variable n
  • Example:-

    class Demo {
        int n = 10; // Instance variable

        public void show(){
            int n = 20; // local varibale

            System.out.println("n = "+n);
            System.out.println("n = "+this.n);
        }

        public static void main(String [] args){
            Demo d = new Demo();

            d.show();
        }
    }

    /*
        Output - 
            n = 20
            n = 10
    */
  • In the above example we can see the local variable n hides the instance variable n when first printed.
  • In other to access the instance variable we use the this operator.

Static members

  • Definition and Usage

    • Static is a keyword used in java to make instance variable or methods static.
    • A static member will have only one instance of it.(i.e all th object of that class will have the same instance of the static method or static variable)
    • In java it is used for:-
      1. It is useful for memory management as the static varibales and methods are allocated only once.
      2. They are used to make a instance variable or method of a class rather than the object of the class.

  • Example


    class StaticDemo {

        static int num = 0;

        StaticDemo(){
            num++;
        }

        public static void show(){
            System.out.println("Num is "+num);
        }

        public static void main(String [] args){
            StaticDemo sd1 = new StaticDemo();
            StaticDemo sd2 = new StaticDemo();

            sd1.show(); // Num is 2
            sd2.show(); // Num is 2
            num++; 
            show(); // Num is 3
        }
    }

Streams

  • Definition and Usage

    • A stream is a sequence of data. In Java, a stream is composed of bytes. It’s called a stream because it is like a stream of water that continues to flow.
    • A stream is a sequence of objects that supports various methods which can be pipelined to produce the desired result.
    • It is used to:-

  • Applet life cycle

    • The time from the start of the applet in the browser to closing of the browser is the life cycle of applet.
    • There are 4 states:-
      • New Born State: The state where the applet is first loaded to the browser and life cycle of applet begins.
        • init() method is called in this state.
        • Most of the initialization is done in this state.
      • Running State: The state when applet is active in the foreground and is interacting with user.
        • start() method is invoked in this state.
        • After start() method run() followed by paint() methods are invoked.
        • User interacts with applet program in this state.
      • Idle State: The state when the page conatining applet is halted temporarily.
        • stop() method is called in this state.
        • After this state either destroy() or start() method is invoked according to the users choice.
      • Dead State: The state when applet program(i.e applet containing web page) is closed.
        • destroy() method is invoked in this state.
        • The destroy() method only gets invoked if the browser is shut down normally.

JDBC and its drivers

  • Definition

    • JDBC Driver is a software component that enables java application to interact with the database.
    • There are 4 types of JDBC Driver:-
      1. JDBC-ODBC bridge Driver: The JDBC-ODBC bridge driver uses ODBC driver to connect to the database. The JDBC-ODBC bridge driver converts JDBC method calls into the ODBC function calls. This is now discouraged because of thin driver.
      2. Native-API Driver: The Native API driver uses the client-side libraries of the database. The driver converts JDBC methods calls into native calls of the database API. It is not written entirely in java.
      3. Network protocal Driver: The Network Protocol driver uses middleware (application server) that converts JDBC calls directly or indirectly into the vendor-specific database protocol. It is fully written in java.
      4. Thin Driver: The thin driver converts JDBC calls directly into the vendor-specific database protocol. That is why it is known as thin driver. It is fully written in Java language.

Connection Pooling

  • Definition and Usage

    • For a database operation to complete, a connection must be established to the database, and the process is a time consuming and expensive process.
    • Therefore Connection pooling has been implemented as a standard for most of the database drivers.
    • The connection pooling creates connection and makes a pool of connection before they are needed.
    • The connection is done in advance and the data search the connection pool and uses the preexisting connection whenever needed.
    • After the completion of task the connection is not terminated but preserved and reused later.

  • Example


    import java.util.*;
    import java.sql.*;

    class ConnectionPoolManager
    {

        String databaseUrl = "jdbc:mysql://localhost:3306/myDatabase";
        String userName = "userName";
        String password = "userPass";

        Vector connectionPool = new Vector();

        public ConnectionPoolManager()
        {
            initialize();
        }

        public ConnectionPoolManager(
            //String databaseName,
            String databaseUrl,
            String userName,
            String password
            )
        {
            this.databaseUrl = databaseUrl;
            this.userName = userName;
            this.password = password;
            initialize();
        }

        private void initialize()
        {
            //Here we can initialize all the information that we need
            initializeConnectionPool();
        }

        private void initializeConnectionPool()
        {
            while(!checkIfConnectionPoolIsFull())
            {
                System.out.println("Connection Pool is NOT full. Proceeding with adding new connections");
                //Adding new connection instance until the pool is full
                connectionPool.addElement(createNewConnectionForPool());
            }
            System.out.println("Connection Pool is full.");
        }

        private synchronized boolean checkIfConnectionPoolIsFull()
        {
            final int MAX_POOL_SIZE = 5;

            //Check if the pool size
            if(connectionPool.size() < 5)
            {
                return false;
            }

            return true;
        }

        //Creating a connection
        private Connection createNewConnectionForPool()
        {
            Connection connection = null;

            try
            {
                Class.forName("com.mysql.jdbc.Driver");
                connection = DriverManager.getConnection(databaseUrl, userName, password);
                System.out.println("Connection: "+connection);
            }
            catch(SQLException sqle)
            {
                System.err.println("SQLException: "+sqle);
                return null;
            }
            catch(ClassNotFoundException cnfe)
            {
                System.err.println("ClassNotFoundException: "+cnfe);
                return null;
            }

            return connection;
        }

        public synchronized Connection getConnectionFromPool()
        {
            Connection connection = null;

            //Check if there is a connection available. There are times when all the connections in the pool may be used up
            if(connectionPool.size() > 0)
            {
                connection = (Connection) connectionPool.firstElement();
                connectionPool.removeElementAt(0);
            }
            //Giving away the connection from the connection pool
            return connection;
        }

        public synchronized void returnConnectionToPool(Connection connection)
        {
            //Adding the connection from the client back to the connection pool
            connectionPool.addElement(connection);
        }

        public static void main(String args[])
        {
            ConnectionPoolManager ConnectionPoolManager = new ConnectionPoolManager();
        }

    }

Socket

  • Definition

    • Java Socket programming is used for communication between the applications running on different JRE.
    • Java Socket programming can be connection-oriented or connection-less.
    • Socket and ServerSocket classes are used for connection-oriented socket programming and DatagramSocket and DatagramPacket classes are used for connection-less socket programming.

  • TCP/IP Communication/ Socket programming


    /*--Client-Side--*/

    import java.net.*;
    import java.io.*;

    class Client {
        public static void main(String [] args){
            try {
                Socket s = new Socket("localhost", 3333);

                DataOutputStream dout = new DataOutputStream(s.getOutputStream());
                DataInputStream din = new DataInputStream(s.getInputStream());
                BufferedReader br = new BufferedReader(new InputStreamReader(System.in));

                String mssg = "";

                while(mssg != "bye"){
                    mssg = din.readUTF();
                    System.out.println("Server says: "+mssg);

                    dout.writeUTF(br.readLine());
                    dout.flush();
                }

                s.close();
                dout.close();
                din.close();

            } catch(Exception e) {
                System.out.println(e);
            }
        }
    }

Multicasting

  • Definition and Usage

    • Multicasting in java is the process of sending data packets to only specific group nodes in the socket communication that are interested to recieve the data packets.
    • Unlike Broadcasting and Unicasting where the data packets are send to either all nodes connected to the network or only a specific node, Multicasting can group specific nodes and send only them the data packets.
    • We use the MultiSocket class for grouping sockets into specific IP’s
    • It is used to create group of nodes in the network to share data.
