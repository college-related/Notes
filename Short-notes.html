RAID

  • RAID (Reduntant Array of Idependent/Inexpensive Disk) is a mechanism or solution for reducing the data fault or data loss during the failure of disk.
  • In this concept we use multiple disks in order to either copy the data across different disks or divide accross the multiple disks.
  • There are 8 lables of RAID from RAID 0 to RAID 6 and a RAID 10
  • Some terms in RAID:
    • Stripping: It is the process of dividing the data into multiple disks equally.
    • Mirroring: It is the process of writing the same data accross multiple devices.
    • Parity: It is the process of rebuilding the data incase of disk faliure, by using a information called as parity.
    • Hot sphere: It is the process of using extra hard disk incase of hard disk failure.
  • The labels of RAID are:-
  1. RAID 0
    - It is the most unused RAID method of all. In this method we use only the stripping mechanism hence there is no fault tolerance at all(i.e if a disk fails then data is lost)
  1. RAID 1
    • It is the most fault tolerant method of all. In this method we use the concept of mirroring hence even when a disk fails all the data are present in another disk. Though the speed of the system is compromised
  1. RAID 2
    • In this method we use the stripping concept as RAID 0 but in bit level.
  1. RAID 3
    • In this method we use the same concept as RAID 2 but with parity.
  1. RAID 4
    • In this method, we use the concept of stripping with parity. Where we have a dedicated disk for storing parity.
  1. RAID 5
    • In this method, we use the conecpt of stripping with parity. It is one of the most used method of RAID by companies, as it gives both speed and fault tolerance. Though it is build for only handling single disk failure. It needs atleast 3 disk.
  1. RAID 6
    • In this method, we use the conecpt of stripping with double parity. It is much more fault tolerant than RAID 5 as it is designed to handle 2 disk failure. Though writing takes alot of time. It needs atleast 4 disks.
  1. RAID 10
    • In this method we use the conept of both RAID 1 and RAID 0 hence the name RAID 1+0. It gives the advantage of both speed and fault tolerance.

DMA

  • DMA (Direct Memory Access) is a input output data transfer device that is used to transfer data from peripheral devices to the memory or viceversa without having to go through the CPU processing.
  • In DMA, the CPU releases the memory bus and those buses are controlled by DMA which allows it to transfer high speed data without having to go through CPU.

  • Process goes as follows:
    • The DMA controller sends a BR(Bus request) to the main processor or CPU
    • The CPU sends back a signal BG(Bus Grant) which means the CPU has released the memory bus
    • Then the I/O Device sends a signal to DMA called DMA Request
    • After which the DMA contoller sends DMA Acknowledge signal back to I/O device which means they are ready to transfer data
    • Then the data is transfered to the memory using the Memory Bus and transfered from memory to I/O device using the I/O Bus

Interrupt handling

  • Interrupt are the signal or request sent by devices to the computer to stop the processor what it is doing and do something else.
  • There are 3 types of interrupt that can occur:-
    1. Hardware interrupt: This is the type of interrupt that is introduced by a hardware device, for example: indicating transfer is complete by devices
    2. Software interrupt: This is the type of interrupt that is produced by programs and codes written by programmers. for example: int 21H, DI, etc
    3. Processor intterupt: This is the type if interrupt that is produced by processor itself. For eaxample: some errors like divide by 0
  • Interrupt are handled by using a interrupt controller
  • The steps involved are:-
    1. I/O devices sends a Interrupt Request signal to the processor
    2. After which the processor stops the current process and save it in a stack and register
    3. Then the processor sends the Interrupt Acknowlegde signal to I/O devices
    4. After that the I/O deivces sends data to the processor
    5. The processor process the data using ISR (Interrupt Service Routine)
    6. Then after the completion of interrupt, the processor goes back to the previous state using the stack and register

Batch System

  • Batch System is the type of OS where there is no direct interaction between user and computer.
  • User simply submits a job to the computer operator and the computer operator places the job on an input device.
  • Then a special program (monitor) manages execution of each program in a batch.
  • There are some disadvantages of using a batch system, which are:
    1. There is no priority program
    2. There is no interaction between user and computer
    3. CPU is often idle

OS services

  • OS services (Operating system services) are the services or functions that a OS is required to accomplish.
  • The functions or responsibiities of an OS differes from computer to computer (in terms of architecture) but the main services/facilities an OS must provide are:-
    1. memory management: It is the action of managing the memory allocation and deallocation in the primary memory of the computer. OS decides which program gets how much memory needed and where the memory is located.
    2. Process management: It is the action of managing all the process that goes to the CPU. It decides which process gets the CPU and when and how long will the process get the CPU.
    3. Device management: It is the action of managing all the device currently connected to the computer. It can be I/O devices or other storage connective device.
    4. Security: Its the OS job to control the security of the computer, protecting the data from unauthorized access.
  • Some other OS services all covers:
    • Job accounting,
    • System performance control
    • error detection, etc

System Calls

  • System calls are the interface between process/program and an operating system. They are available in assembly language instructions. It is usually made when a process in user mode require access to a resource.
  • There are 5 types of system calls:-
    1. Process Control: Deals with process [createProcess(), ExitProcess(), fork()]
    2. File management: Responsible for file manipulation [open(), read(), write()]
    3. Device management: Responsible for device manipulation [readio(), readConsole()]
    4. Information maintenance: Handle information and transfer to OS [settimer(), alarm()]
    5. Communication: Useful for interprocess communication [sleep(), createfilemapping()]

Bad sectors handling in OS

  • Bad sectors in os is defined as the sectors in disk that has gone bad or un usable.
  • It Can be caused by either Physical or Software related Problems or interventions.
  • Physical causes such Fire, laptop drops, etc may cause the hard disk to have bad sector.
  • Software can also cause bad sector when OS fails to read or write in the disk in between working or Processing.
  • There are two methods of handbag bad sector in OS : -
    • Spare sector sparing : In this method the os uses the spare sector (i.e some empty sectors) to remap the bad sectors.
    • Spare sector slipping : In this method the os uses the spare sectors by slipping the positions of the spare sector until the bed sector is moved ahead one step.

  • In this method of file access the order of the file is sequential, i.e if we have to get a file that is in the last position we have to sequential go from the start till the end.

    1. It is slow in search.
    2. We cannot have a feature like skip.

Resource allocation graph

  • Resource Allocation Graph (RAG) is a graphical diagram that represents state of a system in terms of process and resources.
  • It is a graphical representation of the tablular display used in the banker algorithm.
  • The advantage of using RAG is we can detect a deadlock state from the graph easily which cannot be done from the tabular representation
  • Though if the number of processes and resources are more then the RAG representation will be harder to follow.
  • As we know graph has vertices and edges, RAG also has its representation of vertices and edges
  • For vertices:
    • We use the circle for denoting process
    • We use the box/rectangle with single dot inside for denoting the single instance resource
    • We use the box/rectangle with multiple dot inside for denoting the multiple instance resource
  • For edges:
    • If the edge is arrowed from resource to process then it is resource allocation
    • If the edge is arrowed from process to resource then it is resource request

Producers Consumer problem

  • Producers Consumer problem is an classic example of an IPC problem (Inter Process Communication).
  • In this problem we have a producer whose job is to produce resource and fill in the buffer with the produced resources, also we have a Consumer whose job is to consume resource from the buffer, i.e remove the resource from the buffer.
  • The problem lies when the producer starts to produce resources even when the buffer is full hence the produced resources are wasted and not stored anywhere. Also the consumer may start to consume resources from an empty buffer hence not actually doing any work.
  • In order to solve this problem we have to put the producer to sleep when the buffer is full and next time a consumer consumes a resources from the buffer then it notifies the producer to again start producing.
  • Also we make the consumer sleep when the buffer is empty and gets notified when next time a producer produces resources.

Deadlock charactorization

  • Deadlock State is a state in the CPU where more than one process are each waiting for its resources while holding resources that could be used by other process, i.e, all the process in the CPU becomes freezed and hence nothing is happening.
  • It mostly occurs when one process is holding a resource r1 while waiting for resource r2 but another process is holding the resource r2 while holding r1

  • fig: Deadlock State

  • The Characters of Deadlock are:-

    1. Mutual exclusion condition: It means the resources must be non-sharable, i.e only one process can hold the resource at a time
    2. Hold and Wait Condition: Another condition for deadlock is the hold and wait condition, i.e each process must hold the already allocated resources while waiting for additional resources.
    3. Non-preemtive condition: For a deadlock situation to occur, the resources must be Non-preemtive. i.e the resources must not be removed from one process and allocated to another process
    4. Circular Wait Condition: Another important condition is the process and resources must be held in a circular queue manner. i.e one process depend on other resource which is held by other process in a circular fashion.

Fragmentation

  • Fragmentation is an unwanted / unnecessary problem which occur during the allocation of Process in the memory holes avialable.
  • When we allocate the process to any memory hole that can fit the process size then some memory fragments are divided which might be very small for other processes hence when such small memory holes are collected then a large memory is lost. This phenomena is known as the fragmentation.
  • It can handed by a process called as compaction. Compaction is the process of joking the smaller memory fragments into metalogue hole to be used.

  • There are two types or fragmentation :
    • Internal : It is the type of fragmentation that occurs due to the dividing of memory holes when process are allocated and the memory block is divided.
    • External : It is the type of fragmentation in which the memory holes that are unused due to the removal of the process or method.

Demand paging and Thrashing

  • Demand Paging is the process of performing the page swaping function when a process needs a page that is not present in the RAM but in the secondary memory.
  • As the name suggest the demand paging is done only when the demand for the page swaping is requested or else it will not do any swapping.
  • The main point is in the fact that demand paging is done during run time when the process demands for the swapping onlt then we swap the page hence we don’t have to do all the swapping and correcting in advance.
  • Thrashing refers to the high rate of page fault and swapping while processing a process.
  • It is caused by many situations but one of the main situation is when we feed large number of pages to very less number of frames then to accomodate all the pages rapid page fault is encountered and swapping is done which will decrease the CPU utilization.
  • Methods to handling thrashing are:-
    1. Working Set Model: This model states that the demand of frames should not exceed the total number of frames available. If the frames available is exceeded by the demands then there will be thrashing. As long as the demand is met and the process doesnot move from its localitiy(i.e pages involved) there will be no thrashing. Hence this method is used to predict if some process will thrash the system or not then we can prevent.
    2. Page fault Frequency: This is a direct method of handling thrashing where we control the rate of page fault in any process. We can provide a upper limit and lower limit of the page fault rate as shown in the figure below and when the rate drops below lower limit then we can use some of the frames from the process in other process, and when the rate rises above the upper limit we can suspend some processes of the that process.

Polling

  • Polling is a protocol which used check the readyness of any I/0 devices. It checks if any I/0 needs servicing or not by checking the Ready bit of the I/O device.
  • Unlike an interrupt here the CPU checks for the readyness of the I/O device and not the other way around.
  • Where the interrupt can happen at anytime, polling is done at a fixed interval of times.
  • The steps invloved in polling are as follows:
    1. The CPU/Processor at certain interval stops its processing cycle and check for Readyness of I/O devices
    2. If any I/O device has ready bit 1 then the CPU starts to service the deivce.
  • Unlike in interrupt where the service is done by Interrupt Controller here the servicing is done by CPU itself.

Paging

  • Paging is the mechanism of retrieving process from main memory in forms of pages.
  • the process are divided into pages and the main memory is divided into frames. A page table is used to convert the logical address given by CPU to the page address which is also converted into main memory/frame memory.
  • Pages are of fixed size length. size length.
  • It helps reduce external fragmentation but causes internal fragmentation.

Segmentation

  • Segmentation is a memory management technique just like the paging mechanism but instead of dividing into pages it is divided into segments.
  • The segments can be variable size length unlinepages which are of same size.
  • It also uses a table called as segment table to Convert the logical address to the segment address and then to main memory address.
  • It helps to reduce external fragmentation but, causes internal fragmentation.
  • It is smaller in size the table compared to paging.
