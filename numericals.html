Chi - square method

  • STEPS
    1. Calculate the expected values E for each data
    2. Calculate O - E for each data where O is the observed value
    3. Calculate (O-E)2 for each data
    4. Calculate (O-E)2/E for each data
    5. Find X2 as X2 = ∑(O−E)2E\sum \frac{(O-E)^2}{E}∑E(O−E)2​
    6. Find the degree of freedom using (N-1) where N is the total number or classes.
    7. Find significance level (α\alphaα), usually 0.05
    8. Find the Xtabular2X^2_{tabular}Xtabular2​ from the chi square distribution table using degree of freedom and significance level
    9. If X2>Xtabular2X^2 > X^2_{tabular}X2>Xtabular2​ then the null hypothesis is rejected and alternate hypothesis is accepted
    10. Else null hypothesis is accepted

    1. check the α\alphaα value for the degree of freedom and calculated X2X^2X2 in the table
    2. Multiply the α\alphaα with 100% and see if the answer is accepted or not

GPSS program

Command Parameters Meaning Code
GENERATE A,B,C,D,E It is to denote the begining of transaction or system. A: mean time, B: Varaince/Spread time, C: time for first transaction/customer, D: number of transaction, E: priority GENERATE A,B,C,D,E
ORIGINATE A,B It also denote the begining of transaction. The difference with generate is here ORIGINATE A,B
ADVANCE A,B It denote a action being done after going inside the system, example: inspecting, cutting hair, servicing, etc ADVANCE A,B
TRANSFER C It denote a two way path with a condition C, if C is true it goes to the right side else to the left side TRANSFER C, left_path_name, right_path_name
TERMINATE It denote the end of the transaction TERMINATE name
SEIZE It denote the customer/entity has been seized to a factory SEIZE name
RELEASE It denote the customer/entity has been released from factory RELEAZE name
ENTER It denote the customer/entity has enter the storgae ENTER name
LEAVE It denote the customer/entity has left the storage. Difference between factory and storage here LEAVE name
TABULATE It denote a table being created for recording information about the transactions. TABULATE name
MARK It denote a marking point from which the associated table must start recording information. If not defined then the table records from the very begining. MARK name

Generate vs Originate

GENERATE ORIGINATE
It states the transaction is created as long as the program defines. It states the transaction is created for unlimited time.
It has possible parameters, A,B,C,D,E It has only two parameters, A,B
It is represented by a half circle. It is represented by a oval
The mean and spread is written as A,B The mean and spread is written as A:B

Factory vs Storage

Factory Storage
It represents a single server point in the system. It represents more than one server point in the system.
It needs SIEZE and RELEASE to take the item/customer and after service release them. It needs ENTER and LEAVE to make the item/customer enter and leave the storage/service points
It is denoted by a rectangle with a diamond in right side. It is denoted by a rectangle with a circle in right side.
It doesnot need any extra command to determine it is factory. It needs to determine the number of service points using STORAGE command
