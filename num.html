Job Scheduling

  • Every job scheduling has this components:
    • Arrival Time(AT): The time each job arrives at the CPU
    • Brust Time(BT): The time each job takes inside the CPU
  • For Round Robin it also has:
    • Quantum Time(QT): The time each jobs will be processed for
  • For Priority it also has:
    • Priority: Priority given to each job
  • 2nd part of each job scheduling consists:
    • Complete Time(CT): The time for strat to the job completing its processing
    • Turn Around Time(TT): The time for a job to arrive and complete its processing, CT - AT
    • Waiting Time(WT): The total time job to wait in the queue, TT - BT

FCFS (First Come First Serve)

  • Look at the AT
  • Process Jobs in order they arrive
  • 2nd part

SJF (Shortest Job First)

  • Look at the AT
  • If 2 or more jobs encountered
    • Process the job with shortest BT
  • 2nd part

SRTN (Shortest Remaining Time Next)

  • Look at the AT
  • Process the job that arrives first
  • During the process
    • if another job arrives with less BT then current job, process the new job
    • else continue processing the current job
  • Repeat untill all jobs are processed
  • 2nd part

Priority (Pre-emptive)

  • Look at the AT
  • Process the job that arrives first
  • During the process
    • if another job arrives with higher priority then current job, process the new job
    • else continue processing the current job
  • Repeat until all jobs are processed
  • 2nd part

Priority (Non Pre-emptive)

  • Look at the AT
  • Process the job that arrives first
  • If two or more jobs arrives
    • process job with higher priority
  • Repeat until all jobs are processed
  • 2nd part

Round Robin

  • Look at the AT
  • Loop
    • Put the arrived jobs in the queue
    • Process the first job in the queue for QT
    • Put new arrived jobs in the queue
    • If the current job hasn’t finished put back in the queue
  • Until all the process are completed
  • 2nd part

Multilevel queue

  • Each queue has there own jobs
  • Each queue will work independently
  • Later we merge them

Multilevel feedback

  • Each queue will process each job once
  • Then remaining jobs passes on to next queue

Banker algorithm

  • Find Need Matrix = (Max - Allocation)
  • For each process
    • If (Need > available)
      • donot execute the process and go to next process
    • if (available > Need)
      • execute the process
      • Avialable = Avialable + allocation
  • Continue till all process are checked
  • If all the process are processed
    • it is in safe state
  • else
    • it is in deadlock state

Memory allocation

  • Important Note: Once the memory block is used then we can’t use the block and its memory hole again for other unallocated blocks.

First Fit

  • Use the first memory block that fits the unallocated block
  • Then again search from start

Next Fit

  • Use the first memory block that fits the unallocated block
  • Now start from the next memory block

Best Fit

  • Use the first memory block that fits the unallocated block also has the smallest memory hole
  • Then again search from start

Worst Fit

  • Use the first memory block that fits the unallocated block also has the largest memory hole
  • Then again search from start

Page Replacement

  • Page Hit(PH): When the next string reference is already in the frames
  • Page Fault(PF): When the next string reference is not present in the frames and must be replaced

FIFO (First In First Out)

  • Add Strings till frames are available
  • After frames are full
    • If PF, Replace the oldest or first string
  • Continue until all string are replaced

LRU (Least Recently Used)

  • Add Strings till frames are available
  • After frames are full
    • If PF, Replace the string which was used the furthest from current position
  • Continue until all string are replaced

Optimal

  • Add Strings till frames are available
  • After frames are full
    • If PF, Replace the string which will be used the furthest from current position
  • Continue until all string are replaced

Second Chance

  • Add strings till frames are available
  • After frames are full
    • If PH, Give the string reference a second chance bit
    • If PF, Check for the string reference that has no second bit and replace that
      • If there are two or more string references that has no second bit replace the one that is oldest.
    • After every two string reference the second chance bit is removed.

Disk Scheduling

  • Always starts at a Head Position given
  • Has a sequence of position to be read given

FCFS

  • In sequence of given order read each position
  • Seek Time: = sum of all distance travelled

SSTF (Shortest Seek Time First)

  • Read the position which has shortest distance from current Head position
  • Continue till all the position are read
  • Seek Time: = sum of all distance travelled

SCAN / Elevator

  • Given towards higher or smaller
  • If (towards higher)
    • Start scaning towards the last disk position while reading the positions
    • Now move towards the smaller side of disk till the smallest disk position given
  • Seek Time = (Largest pos - initial pos) + (Largest pos - smallest given pos)
  • If (towards smaller)
    • Start scaning towards the first disk position while reading the positions
    • Now move towards the larger side of disk till the largest disk position given
  • Seek Time = (Initial pos - smallest pos) + (Largest given pos - smallest pos)

CSCAN (Circular SCAN)

  • Given towards higher or smaller
  • If (towards higher)
    • Start scaning towards the last disk position while reading the positions
    • Now move towards the smallest disk position without scanning
    • Now move towards the larger side of disk till the remaining given disk positions are read
  • Seek Time = (Largest pos - initial pos) + (Largest pos - smallest pos) + (final pos - smallest pos)
  • If (towards smaller)
    • Start scaning towards the first disk position while reading the positions
    • Now move towards the largest disk position without scanning
    • Now move towards the smaller side of disk till the remaining given disk positions are read
  • Seek Time = (Initial pos - smallest pos) + (Largest pos - smallest pos) + (largest pos - final pos)

LOOK

  • Given towards higher or smaller
  • If (towards higher)
    • Start scaning towards the largest given position while reading the positions
    • Now move towards the smaller side of disk till the smallest disk position given
  • Seek Time = (Largest given pos - initial pos) + (Largest given pos - smallest given pos)
  • If (towards smaller)
    • Start scaning towards the smallest given position while reading the positions
    • Now move towards the larger side of disk till the largest disk position given
  • Seek Time = (Initial pos - smallest given pos) + (Largest given pos - smallest given pos)

CLOOK

  • Given towards higher or smaller
  • If (towards higher)
    • Start scaning towards the largest given position while reading the positions
    • Now move towards the smallest given disk position without scanning
    • Now move towards the larger side of disk till the remaining given disk positions are read
  • Seek Time = (Largest given pos - initial pos) + (Largest given pos - smallest given pos) + (final pos - smallest given pos)
  • If (towards smaller)
    • Start scaning towards the first disk position while reading the positions
    • Now move towards the largest given disk position without scanning
    • Now move towards the smaller side of disk till the remaining given disk positions are read
  • Seek Time = (Initial pos - smallest given pos) + (Largest given pos - smallest given pos) + (largest given pos - final pos)
